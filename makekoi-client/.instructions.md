# コード補完ガイドライン

## 重要な原則

### 既存コードの優先
- **常に既存のメソッド名、クラス名、パターンを優先して提案すること**
- 新しいメソッド名を提案する前に、プロジェクト内の既存コードを検索して類似の実装を探すこと
- 同じ目的の処理が既に存在する場合は、それを再利用すること

### コードベースの一貫性
- このプロジェクト内の既存のコーディングパターンに従うこと
- 他のファイルで使用されている命名規則を踏襲すること
- 新規実装より既存実装の再利用を優先すること

### 補完時の判断基準
1. まずワークスペース内の既存コードを検索
2. 類似の実装パターンを探す
3. 既存のメソッドやプロパティを優先的に提案
4. 新しい名前は既存のものがない場合のみ提案

### 既存クラス

#### システム (`Assets/Scripts/System/`)
| クラス | 役割 |
|--------|------|
| `MonoSingletoneBase<T>` | MonoBehaviourシングルトン基底クラス |
| `ResourceContainer` | スプライトキャッシュ付きリソースロード `GetItemIconSprite()`, `GetSpriteByPath()` |

#### Soundシステム (`Assets/Scripts/System/Sound/`)
| クラス | 役割 |
|--------|------|
| `SoundManager` | BGM/SE管理（静的ファサード） `PlaySE()`, `PlaySeByType()`, `PlayBgm()`, `StopBgm()` |
| `SoundsContainer` | SE/BGMリソース保持 ScriptableObject |
| `BgmFader` | BGMクロスフェード `PlayBgmFade()`, `FadeLow()` |
| `BgmController` | BGMフェードイン/アウト |
| `SeRootController` | SEプール管理 |
| `SeController` | SE1音再生 |
| `LoopSeController` | ループSE音量フェード |

#### UI (`Assets/Scripts/System/UI/`)
| クラス | 役割 |
|--------|------|
| `UIButtonSePlayer` | Buttonクリック時にSEを再生するコンポーネント |

#### 生成済み (`Assets/Scripts/System/Generated/`)
| クラス | 役割 |
|--------|------|
| `CommonSeType` | SEキーのenum（SoundsContainerEditorで再生成） |

#### アーキテクチャメモ
- 非同期処理は **UniTask** を使用
- リアクティブ処理は **R3**（Reactive Extensions）を使用
- UIウィンドウ管理は **AstraydeFramework** の `WindowManager.OpenWindowAsync<T>()` を使用

### 機能追加・バージョンアップの手順

1. **ローカル開発モードに切り替え**
   ```bash
   bash /Users/lilin/projects/AstraydeFramework/use_astrayde_develop.sh
   ```
   → `manifest.json` の `com.astrayde.framework` が `file:` パスに切り替わる

2. **Developリポジトリでブランチを切る**
   ```bash
   cd /Users/lilin/projects/AstraydeFramework/AstraydeFramework_Develop
   git checkout -b feature/xxx
   ```

3. **`Packages/com.astrayde.framework/Runtime/` 以下に機能を実装・コミット**

4. **同期スクリプトを実行**（AssetStore・UPMに反映 + manifest をgit参照に戻す）
   ```bash
   bash /Users/lilin/projects/AstraydeFramework/turn_off_and_sync_to_asset_store_and_upm.sh
   ```
   → スクリプト内の `GIT_REF` のバージョンを先に更新すること

5. **UPMリポジトリでtagを打ってpush**
   ```bash
   cd /Users/lilin/projects/AstraydeFrameworkUPM
   git add . && git commit -m "vX.X.X"
   git tag vX.X.X
   git push && git push --tags
   ```

6. **makekoi-client 側の manifest を新バージョンに更新**して動作確認

### 注意事項
- `Runtime/` 配下のみ同期される（`.meta` / `.asmdef` は除外）
- `Editor/` も同様に同期される
- バージョン番号は同期スクリプトの `GIT_REF` 変数を手動で更新すること
